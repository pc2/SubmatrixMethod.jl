Contribution Guide

Conventions

Source code

  • As indicated by the .JuliaFormatter.toml, we follow the BlueStyle for code formatting.
    • Easiest way to make sure that code is formatted correctly is to use the autoformatting feature of the Julia VSCode extension.
    • Alternatively, you can use JuliaFormatter.jl directly.
  • The user-provided input matrix is always labeled A.
  • The result matrix is always labeled R.

Documentation

Philosophically, we loosly try to implement the ideas put forward by David Laing in the documentation system a.k.a "The Grand Unified Theory of Documentation". TLDR:

  • "References" should only contain technical descriptions (if examples, only very minimal ones)
  • "Explanations" contains methods discussions / descriptions with formulas etc.
  • "Examples" is for How-To guides
  • (We don't really have Tutorials I guess?)
