Testing

ReTest.jl

Sometimes you may want to run only certain @testsets instead of the entire test suite (] test). While Julia's standard library Test doesn't offer such a feature (yet), you can often use ReTest.jl instead. Make sure to install ReTest.jl into your global / default Julia environment such that you can always load it when needed.

Run only certain @testsets

First we need to connect ReTest to SubmatrixMethod.jl as follows.

using SubmatrixMethod, ReTest
ReTest.hijack(SubmatrixMethod)

To get a list of all available testsets, we can use retest(dry=true, verbose=3) which should give something like this

julia> retest(dry=true, verbose=3)
Main.SubmatrixMethodTests
 1| SubmatrixMethod.jl
 2|   Utility
 3|   sprandsymposdef
 4|   Submatrix Construction
 5|   Serial + Threaded
 6|     !inplace
 7|     inplace
 8|     HDF5Input
 9|   IO
10|   Distributed
11|   MPISerial

Let's say we only want to run the "Utility" testset. This can now readily be done via retest("Utility", verbose=3) which gives

julia> retest("Utility", verbose=3)
                               Pass  
Main.SubmatrixMethodTests:
  SubmatrixMethod.jl       |   1027  
    Utility                |   1027

You can also use regex or pass multiple (partial) testset names.

julia> retest(["Utility", "sprand"], verbose=3)
                               Pass  
Main.SubmatrixMethodTests:
  SubmatrixMethod.jl       |   1087  
    Utility                |   1027  
    sprandsymposdef        |     60

See this JuliaCon video and the ReTest documentation for more information on retest and other functionality.

Warning

As of today, ReTest doesn't seem to support testsets with included tests, i.e. include("test_xy.jl"). Hence, you can't use the method above to run the "IO" testset, for example.

Retest + Revise

using Revise, SubmatrixMethod, ReTest
ReTest.hijack(SubmatrixMethod, revise=true)
retest("Utility", verbose=3)
# modify the "Utility" testset, e.g. add a new test
retest("Utility", verbose=3) # will run the modified testset

Automatic live retesting

There is the (experimental) function ReTest.watch that one can use to have ReTest watch changes of certain testsets and automatically retest them. See this JuliaCon video for a demonstration.

using Revise, SubmatrixMethod, ReTest
ReTest.hijack(SubmatrixMethod, revise=true)
ReTest.watch("Utility", verbose=3)
# modifying the "Utility" testset will automatically
# retrigger retesting of this testset

Example (where I first add and then remove a trivial @test true test):

julia> ReTest.watch("Utility", verbose=3)
                               Pass  
Main.SubmatrixMethodTests:
  SubmatrixMethod.jl       |   1027                                       
    Utility                |   1027  

watching...
                               Pass  
Main.SubmatrixMethodTests:
  SubmatrixMethod.jl       |   1028  
    Utility                |   1028  

watching...
                               Pass  
Main.SubmatrixMethodTests:
  SubmatrixMethod.jl       |   1027  
    Utility                |   1027  

watching...
